01Programación clásica vs. ML

Clásica

  • El humano escribe las reglas
  • if / else → código → ejecuta
  • Determinista y auditable

Machine Learning

  • El humano provee datos + arquitectura
  • El entrenamiento descubre las reglas
  • El resultado son pesos (números)

El "código fuente" del comportamiento ya no es texto: son números.

01Qué cambia radicalmente

  • Testing — no lo cubrís con unit tests; necesitás evals sobre distribuciones
  • Debugging — el bug puede estar en los datos, no en el código
  • Versionado — del código + de los datos + de los pesos
  • Reproducibilidad ≠ determinismo

02El modelo es una imagen de Docker

Dockerfile + arquitectura≈Código de entrenamiento
Contexto de build≈Dataset
docker build≈Training run (días · M de USD)
Imagen taggeada (blob inmutable)≈Pesos del modelo
docker run≈Inferencia / serving

02El LLM es un artefacto que no compilás

  • Cientos de GB de pesos
  • Costo de entrenamiento: ~$79M – $100M+ caso GPT-4 — Epoch AI / Sam Altman
  • Lo consumís como dependencia: API o weights de Hugging Face
  • Más parecido a "bajar Postgres" que a "compilar tu app"

03Una red neuronal es un tablero de perillas

  • Función matemática gigante con N parámetros ajustables ("perillas")
  • Una perilla por conexión interna
  • Frontier hoy: 10⁹ – 10¹² parámetros

Random → la salida es basura. La configuración correcta → la predicción que querés. ¿Cómo la encontrás?

03Backpropagation (1986)

  • Predecir → calcular el error (loss)
  • Calcular el gradiente — regla de la cadena: en qué dirección mover cada perilla
  • Ajustar todas un pasito (descenso por gradiente)
  • Repetir millones de veces, con millones de ejemplos
Rumelhart, Hinton & Williams — Nature 323:533–536 (1986)

03Descenso por gradiente, en vivo

  • Soltamos una pelotita en un punto random
  • Gradiente = derivada en ese punto
  • La movemos un pasito en contra del gradiente
  • Converge al mínimo
demos/gradient_descent.py
for _ in range(steps):
    x = x - lr * grad(x)  # un pasito
    # f(x) = (x-3)² + 2
    # grad = 2·(x-3)

Lo mismo, en miles de millones de dimensiones.

04Embeddings: GPS del significado

  • Texto → vector de 384 / 768 / 1536 / 3072 dimensiones
  • Cercanía geométrica = similitud semántica
  • "gato" cerca de "perro"; lejos de "factura"
vec(rey) − vec(hombre) + vec(mujer) ≈ vec(reina)

04Papers fundacionales

  • Word2Vec — Mikolov et al. arXiv:1301.3781 (ene 2013) · NeurIPS 2013
  • GloVe — Pennington, Socher & Manning EMNLP 2014, Doha
  • Primera ola: embeddings estáticos (1 palabra = 1 vector fijo)
  • Hoy: embeddings contextuales (vienen de los Transformers) → resuelven polisemia

052017 — Attention Is All You Need

  • Tira RNN/CNN; deja solo atención
  • Antes: RNN/LSTM secuenciales y con memoria corta
  • Modelo original: 100M params · 41.8 BLEU EN-FR · 3.5 días en 8 GPUs
Vaswani, Shazeer, Parmar, Uszkoreit, Jones, Gomez, Kaiser, Polosukhin — NeurIPS 2017 · arXiv:1706.03762

05Self-attention en una línea

Cada token mira simultáneamente a todos los demás y decide a cuáles prestar atención.

  • Query — "qué información necesito"
  • Key — "esto es lo que yo ofrezco"
  • Value — el contenido que se combina, ponderado por softmax(Q·Kᵀ)
  • Multi-head — varias atenciones en paralelo, cada una mira otra relación

05Por qué fue LA revolución

  • Paralelizable en GPU → modelos mucho más grandes, mucho más rápido
  • Dependencias largas en O(1) hops (vs O(N) en RNN)
  • Token 1 atiende al token 1000 en una sola operación
  • Sin Transformers no había GPT

05Tokenización — cómo entra el texto

  • Texto → IDs numéricos no es 1 palabra = 1 token
  • BPE / tiktoken (o200k_base para GPT-4o+)
  • Costos y context windows se miden en tokens
  • Español se fragmenta más que inglés → pagás más

06De Transformers a LLMs

  • Receta: decoder Transformer + escala + corpus masivo + pre-entrenamiento autoregresivo
  • GPT-3 (175B, may 2020) — primer momento de capacidades emergentes
  • ChatGPT (30 nov 2022, GPT-3.5) — producto masivo
  • GPT-4 (14 mar 2023) · Claude Opus 4.8, GPT-5.1, Gemini 3 Pro (2025–26)

"No hay magia adicional. La magia está en la escala."

06Scaling laws

  • Kaplan et al. 2020 — la loss escala como ley de potencia en params, datos y cómputo arXiv:2001.08361
  • Chinchilla (Hoffmann et al. 2022) — el óptimo es ~20 tokens / parámetro arXiv:2203.15556 · 70B sobre 1.4T tokens > Gopher 280B
  • Implicancia: el problema se vuelve presupuestario

06El LLM es un sistema operativo

  • Es infraestructura
  • Lo construyen 5–10 organizaciones en el mundo
  • Vos lo consumís
  • Lo configurás con prompts, system messages, herramientas, contexto — no lo recompilás

07De LLM a agente

LLM puro

  • texto → texto
  • No corre código, no lee tu DB, no abre PRs
  • No itera si se equivoca

Agente

  • LLM en loop con herramientas + objetivo
  • Planifica → ejecuta → observa → itera
  • Repite hasta resolver la tarea

07Patrones canónicos

  • ReAct — Thought → Action → Observation Yao et al., arXiv:2210.03629 · ICLR 2023
  • Toolformer — el modelo aprende a invocar APIs, self-supervised Schick et al., NeurIPS 2023 (oral)
  • Function calling — hoy estándar en todos los APIs de modelo

07Model Context Protocol (MCP)

  • Anthropic, 25-nov-2024 — "USB-C para AI" D. Soria Parra & J. Spahr-Summers
  • Donado a la Linux Foundation / Agentic AI Foundation (9-dic-2025)
  • Co-fundadores: Anthropic, Block, OpenAI · Platinum: AWS, Bloomberg, Cloudflare, Google, Microsoft
  • 97M+ descargas/mes de SDKs · 10.000+ servidores activos

07Workflows vs. agents

  • Workflows — caminos predefinidos en código
  • Agents — el LLM dirige dinámicamente su propio proceso
  • Es un gradiente de autonomía
  • Recomendación: empezá simple. Escalá complejidad solo cuando hace falta
Anthropic — "Building Effective Agents" (19-dic-2024)

·Qué cambia para MLOps

  • El "modelo" deja de ser algo que vos compilás
  • El "deploy" incluye tools, permisos, sandboxing
  • El "monitoreo" incluye evals, traces, cost por request
  • IaC para herramientas y permisos de agentes · CI/CD para prompts y agent configs
  • Tu rol se mueve hacia arriba en el stack → próximo módulo: agentes en serio

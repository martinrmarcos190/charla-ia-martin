Dos caminos, un taller

Elegí tu camino

A Claude Code + Spec Kit

CLI de Anthropic que corre en tu terminal. Spec Kit le agrega el flujo SDD con artefactos .specify/ y comandos /speckit.*

B Kiro de AWS

IDE basado en VS Code con SDD integrado de fábrica: specs, steering y hooks viven nativamente en la barra lateral. Sin Spec Kit extra.

El 90% del taller es idéntico en ambos caminos. La diferencia está solo en cómo creás las specs y cómo invocás el agente.
AMBOS Paso previo

Base común — verificar antes

uv --version
python3 --version   # necesitás ≥ 3.11
git --version
docker --version
Salida esperada:
uv 0.x.x  ·  Python 3.11.x o superior  ·  git version 2.x  ·  Docker version 2x.x
Si alguno falla: brew install uv git docker (macOS) o bajá el instalador oficial.
Python < 3.11: uv python install 3.11
A Paso 1 de 5

Instalar Claude Code

Instalador nativo recomendado por Anthropic (sin Node)

macOS / Linux

curl -fsSL https://claude.ai/install.sh | bash

Windows PowerShell

irm https://claude.ai/install.ps1 | iex
Alternativa legada (requiere Node 18+, nunca con sudo):
npm install -g @anthropic-ai/claude-code
El instalador descarga el binario, lo deja en ~/.local/bin/claude y configura auto-actualización.
A Paso 2 de 5

Verificar Claude Code

claude --version
claude doctor
Salida esperada:
claude --version devuelve algo como 1.x.x
claude doctor muestra todos los checks en verde (autenticación, dependencias, red)
Si claude no se encuentra: abrí una terminal nueva o ejecutá
source ~/.zshrc (macOS/Linux) para que el PATH se actualice.
A Paso 3 de 5

Instalar Spec Kit

# Instalación persistente con uv
uv tool install specify-cli --from git+https://github.com/github/spec-kit.git

# Alternativa efímera (sin instalar)
uvx --from git+https://github.com/github/spec-kit.git specify init <PROJECT>

Verificar después:

specify check
specify version
Salida esperada: ambos comandos responden sin error con versión y estado del proyecto.
A Paso 4 de 5

Crear repo y configurar SDD

mkdir taller-sdd && cd taller-sdd
git init
claude          # primera vez: abre navegador para autenticar OAuth

Requiere cuenta Pro / Max / Team / Enterprise. Una vez dentro de Claude Code:

/init                                  # genera CLAUDE.md
specify init . --integration claude    # OJO: --ai quedó deprecado
Salida esperada de specify init: crea el directorio .specify/ y los comandos en .claude/commands/
A Paso 5 de 5

Confirmar que todo está listo

Dentro de Claude Code, tipear / y verificar que aparecen los /speckit.*

En la terminal:

ls .specify/
ls .claude/commands/
Deberías ver:
.specify/ con archivos de configuración del proyecto
.claude/commands/ con los archivos speckit.*.md
En Claude Code: el menú / lista /speckit.create, /speckit.list, etc.
B Paso 1 de 2

Instalar y loguear Kiro

B Kiro — IDE con SDD nativo

No instalás Spec Kit ni Claude Code. De la base común solo necesitás uv, Python, Git y Docker (para el Bloque 4).

  1. Descargar desde kiro.dev/downloads — disponible para macOS, Linux y Windows
  2. Instalar como cualquier aplicación (arrastrá a Applications en macOS)
  3. Al abrir: loguear con Google, GitHub, Builder ID o AWS SSO
No hace falta cuenta de AWS. Google o GitHub alcanzan para arrancar.
B Paso 2 de 2

Abrir carpeta y confirmar SDD

En Kiro:

  1. File → New Folder → crear taller-sdd → Open Folder
  2. Opcional — desde la terminal integrada: git init
Qué deberías ver:
Panel "Specs" visible en la barra lateral izquierda
Botón "+" debajo de Specs para crear la primera spec
El chat de Kiro responde en el panel lateral derecho
Si no ves el panel Specs: revisá que abriste una carpeta (no un archivo suelto). Usá File → Open Folder.
Antes de seguir

Checklist final por camino

A Claude Code + Spec Kit

  • uv, claude y specify responden en la terminal
  • Veo el directorio .specify/ en el repo
  • Veo .claude/commands/ con los archivos speckit
  • Tipeo / en Claude Code y aparecen los /speckit.*
  • claude doctor muestra todo en verde

B Kiro

  • Kiro instalado y con sesión activa
  • Abrí una carpeta (taller-sdd) con Open Folder
  • Veo el panel Specs en la barra lateral
  • El botón "+" bajo Specs funciona
  • uv, python3 ≥ 3.11, git y docker responden
Si algo falla

Troubleshooting rápido

claude: command not found
Abrí una terminal nueva o ejecutá source ~/.zshrc.
El binario queda en ~/.local/bin/claude — verificá que ese path esté en tu $PATH.
No aparecen los /speckit.* en Claude Code
Chequeá que estés en la carpeta correcta del repo.
Reiniciá Claude Code y verificá con ls .claude/commands/.
Si usaste --ai en vez de --integration claude, corré specify init . --integration claude de nuevo.
Kiro no muestra el panel Specs
Seguramente abriste un archivo suelto. Cerrá y reabrí con File → Open Folder apuntando a la carpeta del proyecto.

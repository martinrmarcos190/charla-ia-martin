1.1 · El cambio de paradigma

De reglas a patrones

Software tradicional

Programás el qué hacer

if mail.tiene("ganaste $$$"): marcar_spam() elif remitente in lista_negra: bloquear()

El programador sabe y dicta las reglas.

Machine learning

Programás el cómo aprender

datos = [(mail, "spam"), (mail, "ok"), ...] model.fit(X, y) # las reglas las descubre el modelo

Le das ejemplos y descubre las reglas solo.

  • Las reglas no escalan: "reconocé un gato", "esto es spam", "traducí esto"
  • Costo del paradigma: opacidad · no determinismo · dependencia de los datos

1.2 · ¿Qué es un modelo?

Un modelo entrenado es un artefacto compilado

Viene de la estadística (regresión = años 50–60). Cambió la escala, no la idea. Clásicos vigentes: regresión · árboles · Random Forest · gradient boosting · SVM.

1.3 · Redes neuronales

Backpropagation: un CI loop con auto-fix

Una red = activación(W·x + b) apilado en capas. Aprenden las features solas → adiós feature engineering manual.

1. Pesos en valores random
2. Predigo un ejemplo (sale mal)
3. Mido el error (loss)
4. Propago el error hacia atrás (gradiente)
5. Ajusto cada peso un poquito
6. Repito ×millones

Analogía: un CI loop infinito donde el test es "qué tan lejos estuvo la predicción del label" y el commit --amend automático ajusta los pesos.

Rumelhart, Hinton & Williams (1986). Learning representations by back-propagating errors. Nature 323:533–536.

1.4 · El puente hacia los LLMs

Embeddings: coordenadas GPS del significado

  • One-hot: cada palabra ortogonal → no captura significado (perro y gato tan lejos como perro y factura)
  • Embedding: vector denso (100–3072 dim) donde proximidad = similitud semántica
  • vector(rey) − vector(hombre) + vector(mujer) ≈ vector(reina) — aritmética sobre significado
  • Métrica de cercanía: similitud coseno (1 = igual · 0 = sin relación)
Buenos Aires está cerca de Montevideo y lejos de Tokio en el mapa físico.
"perro" está cerca de "gato" y lejos de "factura" en el mapa semántico.

Base de: RAG · búsqueda semántica · recomendadores · deduplicación · detección de fraude.

Mikolov et al. (2013), arXiv:1301.3781 (Word2Vec) · Pennington et al. (2014), EMNLP (GloVe).

1.5 · La revolución

Transformers y atención

  • Pre-2017: RNN/LSTM → secuenciales, pierden contexto largo, no paralelizan
  • 2017: Attention Is All You Need (Vaswani et al., Google) tira las RNN a la basura
  • Idea: cada token mira a todos los demás simultáneamente y decide a cuáles prestar atención
Query (qué busco) · Key (qué ofrezco) · Value (qué tengo) → softmax(Q·Kᵀ / √dₖ)·V
Multi-head: varios en paralelo, cada cabezal aprende relaciones distintas (sintaxis, semántica, correferencia).

Por qué fue LA revolución: paraleliza (destraba la escala) + captura contexto largo.

Análogo MLOps: RNN = build secuencial (Jenkins viejo) · Transformer = build masivamente paralelo (Bazel/Buildkite).

Vaswani et al. (2017). Attention Is All You Need. arXiv:1706.03762 (NeurIPS 2017).

1.6 · La conexión completa

De Transformers a LLMs

Receta: Transformer decoder-only → escalar (GPT-3 = 175B params) → entrenar para predecir el próximo token sobre medio internet → fine-tuning + RLHF.

Un LLM es un autocomplete cósmico.

  • Tokenización: el modelo ve enteros, no strings · cl100k_base (GPT-4) · o200k_base (GPT-4o, GPT-5, o1/o3/o4)
  • ⚠️ NO ES DETERMINÍSTICO: en vez de elegir el token más probable, muestrea de una distribución (temperature, top_p, top_k)
prompt → [Transformer] → distribución P(token) sobre el vocabulario → muestreo → token → (repetir)
Brown et al. (2020), arXiv:2005.14165 (GPT-3) · Devlin et al. (2018), arXiv:1810.04805 (BERT).

1.7 · Lo más importante para el resto del curso

De LLM a Agente

LLM solo: genera texto. Un agente: planifica, usa tools, ejecuta acciones, observa, itera.

  • Anthropic (2024): workflows (camino de código predefinido) vs agents (el LLM dirige su propio proceso)
  • Bloque mínimo: augmented LLM = LLM + retrieval + tools + memory
Yao et al. (2022), arXiv:2210.03629 (ReAct) · Anthropic (2024), Building Effective Agents.
